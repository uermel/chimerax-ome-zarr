Command: open

Usage: open ( filename.zarr | ngff:[protocol:]URL.zarr ) [ scales scale1,scale2,... ]

OME-Zarr files can be opened with the open-command similar to any other ChimeraX-supported file. When appending the "ngff:"-prefix to the URL and including a protocol prefix (such as "s3://" or "smb://") allow loading Zarr files from remote storage. When no scales are specified, this command will create a single Volume per Zarr-file that will stream and cache data on demand (multiscale level determined by the "step" setting of the volume command or volume viewer. When scales are specified, the same number of Volumes will be loaded simultaneously, each containing one of the multiscale levels.

Examples:

open /path/to/image.zarr - open a local OME-Zarr
open ngff:s3://bucket/path/to/image.zarr - open an OME-Zarr from AWS S3
open ngff:s3://bucket/path/to/image.zarr scales 1,2 - open two multiscale levels of an OME-Zarr from AWS S3
